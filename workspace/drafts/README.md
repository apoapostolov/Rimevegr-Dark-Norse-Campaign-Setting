# Drafts

Unapproved chapter drafts, scene fragments, and experimental
prose that did not pass the review cycle. Stored here for
potential revisit — rejected today does not mean rejected forever.

## Structure

```text
drafts/
├── ch14/
│   ├── v1_scene2_negotiation.md
│   ├── v2_scene2_negotiation.md
│   └── v1_scene3_vote_rejected.md
├── ch16/
│   └── v1_barrow_combat_choreographer_raw.md
└── fragments/
    ├── kell_monologue_unused.md
    └── ashenmoor_description_alt.md
```

## Naming Convention

Chapter drafts: `vN_sceneN_description.md`
Fragments not tied to a chapter: placed in `fragments/`.

## Draft Header

Every draft file begins with:

```markdown
# Draft: [Description]

**Chapter:** [N or "fragment"]
**Version:** [N]
**Author:** [which agent produced this]
**Date:** YYYY-MM-DD
**Status:** SUPERSEDED | REJECTED | PARKED | EXPERIMENTAL
**Reason:** [why it was not approved]
**Notes:** [what might be salvageable]
```

## Statuses

- **SUPERSEDED:** A better version replaced this draft.
- **REJECTED:** Failed review. Reason recorded.
- **PARKED:** Not rejected — tabled for later consideration.
  May contain good ideas that do not fit current chapter.
- **EXPERIMENTAL:** Deliberate experiment with voice, structure,
  or approach. Not intended for direct use but as reference.

## Rules

- Never delete drafts. They are creative inventory.
- Author may mine old drafts for phrases, details, or ideas.
- Editor references rejected drafts when the same mistake
  recurs: "See `drafts/ch14/v1_scene2` — same em-dash problem."
- Choreographer stores raw combat narratives before Author
  integration.
- Fragments folder is for orphaned prose that might find a home.
